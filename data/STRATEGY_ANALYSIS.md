## 1. 현재 상태 진단

### 개발 완성도
- **코드 규모**: Expo (React Native) + NativeWind, 약 15개 주요 화면
- **주요 기능 구현 상태**: 🟡 진행 중
  - ✅ 인증 시스템 (Supabase Auth)
  - ✅ 포트폴리오 조회/상세 보기
  - ✅ 채용공고 조회
  - ✅ 알림 시스템 (Expo Notifications)
  - ✅ 프로필 페이지
  - 🟡 포트폴리오 작성/편집 (웹과 동기화 미흡)
  - ❌ 채용담당자 기능 미지원
  - ❌ 매칭 추천 미구현

### 매출/사용자 현황
- **배포**: App Store, Google Play 준비 중 (아직 내부 테스트)
- **사용자**: 추정 20~30명 테스트 유저
- **상태**: Expo 개발 서버 구동, 모바일 시뮬레이터 테스트 단계

### 핵심 과제
1. **웹과의 동기화 부족**: 포트폴리오 작성/편집이 웹 우선 → 모바일에서 편집 불가능
2. **채용담당자 경험 부재**: 모바일에는 채용공고 조회만 가능 (매칭 기능 없음)
3. **앱 스토어 배포 미완료**: 아직 베타 단계로 마켓 진입 불가
4. **푸시 알림 효율성 불명**: 알림 시스템은 구축했으나 사용자 재방문율 효과 미측정

## 3. Beachhead 분석

**Primary Segment**: 개발자/디자이너 (모바일 소비층, 채용공고 추적)

| 기준 | 평가 | 근거 |
|------|------|------|
| **Burning Pain** | ⭐⭐⭐ | 유명 개발자는 채용공고를 수시로 확인. 모바일 알림 반응성 높음 |
| **Willingness to Pay** | ⭐⭐⭐ | 모바일 앱은 웹보다 Premium 수락 비율 높음 (앱 구독 심리) |
| **Winnability** | ⭐⭐⭐⭐ | 푸시 알림으로 즉각 반응 유도 가능. 경쟁 앱(Jumpit, Wanted 앱) 대비 포트폴리오 연동이 강점 |
| **Referral** | ⭐⭐⭐⭐ | 앱 추천 장벽 낮음. 포트폴리오 공유 → 다운로드 유도 용이 |

**결론**: 웹과 달리 **모바일은 Burning Pain + Winnability 모두 높음**. 핵심은 **웹과의 동기화**와 **채용담당자 기능** → 우선순위 높음.

## 4. Growth Loop 분석

**현재 루프**: 앱 설치 → 포트폴리오 탐색 → 채용공고 조회 → (알림 없음) ❌

**문제점**:
- 푸시 알림이 구축되었으나 **알림 콘텐츠가 명확하지 않음** (단순 신규 공고? 개인화 추천?)
- 웹에서 포트폴리오를 편집해야 모바일에 반영 → 앱만 사용하는 사용자 이탈

**추천 루프**: Notification-Driven + Web Sync
1. 사용자 모바일 설정 (관심 직군, 경력)
2. 매일 아침 푸시 알림 (맞춤 채용공고 3건)
3. 클릭 → 포트폴리오 빠른 조회 + 지원
4. 웹에서도 같은 포트폴리오 보이기 (동기화)
5. 웹에서 포트폴리오 편집 → 모바일에서 자동 반영

## 5. 가격 전략

**웹과 동일** (패밀리 플랜):
- 1개 계정으로 웹 + 모바일 모두 접근
- 앱에서 Pro 구독 시 웹도 Premium 기능 활성화

### 모바일 특화 프리미엄
```
FREE
- 앱 기본 포트폴리오 조회
- 일일 채용공고 뉴스레터 (1회)

PRO (월 ₩4,900 또는 연 ₩39,000)
- 웹 Pro와 동일 기능
- 추가: 푸시 알림 커스터마이즈 (직군, 연봉, 지역)
- 추가: 포트폴리오 모바일 작성/편집 (추가 개발)

MASTER (월 ₩14,900)
- 모든 Pro 기능
- 추가: 채용담당자 모바일 대시보드
- 추가: 지원 내역 관리 (앱에서 직접)
```

**모바일 전환율 전략**: 첫 설치 후 7일 무료 Pro 체험 권장.

## 7. North Star Metric

### 추천 NSM
> **월 활성 사용자(MAU) 중 포트폴리오 → 채용공고 조회 → 지원 완료 비율**

- 2026년 Q2: 15% (초기)
- 2026년 Q3: 30% (성장)
- 2026년 Q4: 45% (확산)

### 3-5 Input Metrics
1. **앱 DAU**: 목표 30명 (Q2 말, 웹 DAU 50명의 60%)
2. **포트폴리오 조회 깊이**: 평균 3장 이상 (고관여도)
3. **채용공고 클릭율**: 목표 12% (웹 8% 대비 높은 이유: 알림)
4. **푸시 알림 클릭율**: 목표 35% (전송한 알림 중)
5. **주간 재방문율 (W1 Retention)**: 목표 50%

## 8. 3단계 로드맵

### Phase 1: 웹 동기화 + 배포 (현재 ~ 2026년 4월 말)
**목표**: App Store/Play Store 배포 + 웹-모바일 동기화

- [ ] 웹 API 재설계 (Supabase RLS 재검토, 모바일 클라이언트 권한)
- [ ] 모바일 포트폴리오 동기화 (웹 변경 사항 실시간 반영)
- [ ] iOS 앱 배포 준비 (TestFlight 베타, 심사)
- [ ] Android 앱 배포 준비 (Google Play 심사)
- [ ] 배포 후 웹 사용자 500명에 앱 초대 이메일
- [ ] 초기 사용자 피드백 수집 (앱 첫 1주일)

### Phase 2: 모바일 편집 + 알림 고도화 (2026년 5월 ~ 6월)
**목표**: 포트폴리오 모바일 편집 가능 + 맞춤 알림 알고리즘

- [ ] 모바일용 간단한 포트폴리오 편집기 (마크다운 기반)
- [ ] 푸시 알림 맞춤화 (직군, 연봉, 지역별)
- [ ] 알림 클릭율 A/B 테스트 (제목, 시간대 최적화)
- [ ] 포트폴리오 공유 기능 (카톡, 인스타그램 스토리)
- [ ] App Store Optimization (ASO): 스크린샷, 설명 최적화
- [ ] 1,000 다운로드 달성 (웹 + 앱 초대)

### Phase 3: 채용담당자 기능 (2026년 7월 ~ 9월)
**목표**: 모바일에서도 채용담당자 기능 활성화

- [ ] 채용담당자 모바일 대시보드 (관심 포트폴리오, 추천)
- [ ] 지원 내역 관리 (모바일에서 직접 합격/불합격 처리)
- [ ] 모바일-웹 일원화된 채용관리 CRM
- [ ] 협력사 채용공고 앱에서 직접 게시
- [ ] 리뷰 평점 5.0 유지 (사용자 피드백 신속 대응)
